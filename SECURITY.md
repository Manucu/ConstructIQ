# Security Policy

Thank you for helping keep ConstructIQ secure.

The security of our users, customers and data is a top priority.

---

# Supported Versions

| Version | Supported |
|----------|-----------|
| 1.x | ✅ Yes |
| 0.x | 🚧 Development Only |

Only the latest stable version receives security updates.

---

# Reporting a Vulnerability

If you discover a security vulnerability, **please do not create a public GitHub Issue.**

Instead, report it privately to the project maintainer.

Include as much information as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested mitigation (if known)
- Screenshots or logs (if applicable)

Responsible disclosure helps protect users while the issue is investigated and resolved.

---

# What to Report

Please report vulnerabilities involving:

- Authentication
- Authorization
- Permission bypass
- JWT handling
- SQL Injection
- Cross-Site Scripting (XSS)
- Cross-Site Request Forgery (CSRF)
- File Uploads
- Sensitive Data Exposure
- API Security
- Business Logic vulnerabilities
- Audit Log manipulation
- Financial Data exposure

---

# Response Process

When a vulnerability is reported, the following process is followed:

1. Acknowledge the report.
2. Validate the issue.
3. Assess the impact.
4. Develop a fix.
5. Test the solution.
6. Release the security update.
7. Credit the reporter (if desired).

---

# Scope

This policy applies to:

- Source Code
- Backend API
- Frontend Application
- Database Design
- Authentication
- Authorization
- File Storage
- Documentation

---

# Security Principles

ConstructIQ follows these principles:

- Security by Design
- Least Privilege
- Defense in Depth
- Secure Defaults
- Documentation First
- Auditability
- Confidentiality
- Integrity
- Availability

---

# Confidential Information

The following information is considered confidential:

- Worker Rates
- Internal Costs
- Company Financial Data
- Supplier Financial Information
- JWT Secrets
- Environment Variables
- Database Credentials

Unauthorized access or disclosure should be reported immediately.

---

# Third-Party Dependencies

ConstructIQ regularly updates dependencies to address known vulnerabilities.

Dependencies should be reviewed before every release.

---

# Security Updates

Security fixes are released as soon as possible after validation.

Critical vulnerabilities take priority over feature development.

---

# Responsible Disclosure

Please allow reasonable time to investigate and resolve reported vulnerabilities before making them public.

ConstructIQ appreciates responsible security research.

---

# Contact

For security-related matters, please contact the project owner privately.

Security reports should never contain confidential customer information unless absolutely necessary.

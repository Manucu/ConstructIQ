# Code of Conduct

> **Build once.**
>
> **Record once.**
>
> **Trust forever.**

---

# Purpose

ConstructIQ is committed to creating a welcoming, respectful and professional environment for everyone who contributes to the project.

This Code of Conduct defines the expected behavior for all contributors and helps ensure positive collaboration.

---

# Our Values

ConstructIQ is built on the following values:

- Respect
- Professionalism
- Transparency
- Collaboration
- Responsibility
- Continuous Learning

Every contributor is expected to uphold these values.

---

# Expected Behavior

Contributors are expected to:

- Treat everyone with respect.
- Be constructive during discussions.
- Accept feedback professionally.
- Help maintain a positive environment.
- Focus on solving problems rather than assigning blame.
- Document important decisions.
- Follow the project's coding and documentation standards.

---

# Unacceptable Behavior

The following behaviors are not tolerated:

- Harassment or discrimination of any kind.
- Offensive or abusive language.
- Personal attacks.
- Intentionally disruptive behavior.
- Publishing confidential information.
- Deliberately introducing malicious code.
- Ignoring project security standards.

---

# Communication

ConstructIQ encourages:

- Clear communication.
- Constructive feedback.
- Evidence-based technical discussions.
- Respectful disagreement.

Technical decisions should always prioritize the long-term quality of the project.

---

# Documentation First

ConstructIQ follows a Documentation First philosophy.

Before implementing significant functionality:

- Review existing documentation.
- Update documentation if business rules change.
- Implement the documented behavior.

Documentation is considered part of the product.

---

# Security Responsibility

Every contributor shares responsibility for protecting project data.

Never:

- commit secrets
- expose credentials
- bypass permission checks
- weaken security without discussion

Security is everyone's responsibility.

---

# Code Quality

Every contribution should:

- follow coding standards
- be readable
- be tested
- be documented
- respect existing architecture

---

# Review Process

Code reviews should focus on:

- correctness
- readability
- maintainability
- security
- business requirements

Feedback should always be respectful and constructive.

---

# Reporting Issues

If you encounter:

- inappropriate behavior
- security vulnerabilities
- confidential data exposure

Please report the issue privately to the project maintainer.

---

# Enforcement

Project maintainers reserve the right to:

- request changes
- reject contributions
- remove inappropriate content
- restrict participation when necessary

These actions aim to protect the project and its contributors.

---

# Acknowledgment

By contributing to ConstructIQ, you agree to follow this Code of Conduct and help maintain a professional and respectful community.

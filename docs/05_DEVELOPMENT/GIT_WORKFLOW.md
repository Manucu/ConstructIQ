## Purpose

This document defines how Git and GitHub are used in ConstructIQ.

The goal is to keep the repository clean, organized and easy to understand.

---

## Core Principles

- Commit often.
- Commit complete logical changes.
- Use clear commit messages.
- Never commit broken code intentionally.
- Push after important milestones.
- Keep documentation and code synchronized.

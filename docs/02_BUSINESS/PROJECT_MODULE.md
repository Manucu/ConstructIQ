# ConstructIQ Project Module

## Purpose

The Project Module is the core of ConstructIQ.

It represents the digital workspace of a construction project, where users can manage progress, stages, budget, expenses, workers, site diary, photos, documents and reports.

Each project acts as a digital twin of the real construction site.

---

## Main Goal

The goal of this module is to give owners, project managers, site engineers and clients a complete view of a construction project in one place.

---

## Project Overview

Each project contains:

- general project information
- project status
- progress percentage
- project health score
- budget information
- active stages
- recent expenses
- site diary entries
- uploaded photos
- documents
- reports

---

## Project Details Page

The Project Details page will include the following sections:

### 1. Overview

Displays the general state of the project.

Includes:

- project name
- client
- address
- status
- progress
- project health
- start date
- deadline

---

### 2. Stages

Displays all construction stages.

Examples:

- Planning
- Permits
- Site Organization
- Excavation
- Foundation
- Structure
- Masonry
- Roof
- Installations
- Finishes
- Reception

Each stage contains:

- status
- progress
- planned budget
- spent amount
- start date
- end date

---

### 3. Budget & Expenses

Tracks project costs.

Includes:

- total budget
- spent amount
- remaining budget
- budget by stage
- expenses by category

Expense categories:

- materials
- labor
- equipment
- transport
- other

---

### 4. Site Diary

Stores daily site reports.

Each diary entry contains:

- date
- weather
- workers count
- worked hours
- activities
- observations
- related stage
- uploaded photos
- daily expenses

---

### 5. Photos

Stores construction site images.

Each photo contains:

- project
- stage
- diary entry
- upload date
- uploaded by
- description

Photos can be filtered by:

- date
- stage
- uploader

---

### 6. Documents

Stores project documents.

Examples:

- contract
- invoices
- architectural plans
- structural plans
- permits
- reports
- reception documents

---

### 7. Reports

Generates project reports.

Examples:

- daily report
- weekly report
- budget report
- progress report
- final project report

---

## Project Health

Each project has a Project Health score from 0 to 100.

The score is calculated based on:

- budget control
- progress
- schedule performance
- documentation
- site activity
- risk level

Example:

Project Health = 91 / 100

---

## User Roles

### Owner

Can view and manage all project data.

### Project Manager

Can manage project stages, tasks, budget and reports.

### Site Engineer

Can create diary entries, upload photos and register daily site data.

### Accountant

Can manage invoices, expenses and budget information.

### Client

Can view project progress, photos, documents and reports.

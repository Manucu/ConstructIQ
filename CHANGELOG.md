# Changelog

All notable changes to ConstructIQ will be documented in this file.

The format is based on **Keep a Changelog** and follows **Semantic Versioning (SemVer)**.

---

## Versioning

ConstructIQ uses Semantic Versioning.

Format:

```text
MAJOR.MINOR.PATCH
```

Example:

```text
1.0.0
```

Where:

- **MAJOR** → Breaking changes
- **MINOR** → New features
- **PATCH** → Bug fixes
